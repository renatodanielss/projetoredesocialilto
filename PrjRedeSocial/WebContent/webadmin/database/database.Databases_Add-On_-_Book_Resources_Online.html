Booking application, which enables visitors/users to search for and book available resources. Enables registered visitors/users to view their bookings and website administrators to manage bookings.

Resource Categories and Resources

Please note that when you have imported the application the first thing you need to do is to add your resource categories and resources to custom databases. A resource category could for example be "Single Room" and a resource could for example be "Room 204". If you enter "Room" in the Resource Type field below then resource categories should first be added to the "Room Categories" custom database and then actual resources can be added to the "Room Resources" custom database. When you have ensured that you have the right resource categories and resources in the custom databases the booking application is ready for use.

Start Page

If you for example enter "Room" in the Resource Type field below the name of the start page for your application is "Room Booking" and the page is located in the page group with the name "Room Bookings". The start page will provide you with more details on how to use the application. The application will store bookings in the "Room Bookings" table.

Resource Type

Enter a type name for your resources in the Resource Type input field below. You could for example enter "Room", "Laptop" or "Car". If you enter "Room" then all the pages for your room booking application will be located in the "Room Bookings" page group and the start page for your booking application is the "Room Booking" page.

Time Unit

You can book resources spanning hours, nights or days. If you select Hour then resources can be booked for one or several hours and will be priced per hour. If you select Night then resources can be booked between two dates and will be priced by the number of nights between the two dates. If you select Day then resources can be booked for one or several days, and will be priced by the number of days selected.

Default Capacity

The booking module enables you to assign a capacity to each resource category. If you have two resource categories, which are single rooms for 1 adult and double rooms for 2 adults the capacity for the single rooms would be 1 and the capacity for the double rooms would be 2. Enter the default capacity your visitors/users should see when booking.

Access to Book

Access to book resources can be completely public or for registered visitors/users only. If you want access to book resources to be public please select Public below. If you want access to book resources to be for registered visitors/users only please select Private below.

Resource Selection

You may have several resources in each resource category. For example you may have "Room1" and "Room2", which are both in the "Single Room" resource category and "Room3", which is in the "Double Room" category. Please select if visitors/users should be able to select specific resources (e.g. "Room2") or resource categories (e.g. "Single Room").

View Bookings

The application includes pages, which enable administrators to view and manage bookings. In addition you can choose to include a page, which enables any registered visitor/user to view existing bookings. If you wish to include this page please select Any Registered Visitor/User below.

Advanced Access Restrictions

If you are using the Small Business Edition or the Professional Edition of the web content management system you can easily change the access restrictions for the booking application after it has been imported. The application's pages are divided into various page types and access restrictions for the corresponding content types can be modified. The actual booking pages are located in the "Room Booking" page type. The pages that enable users to manage their bookings are located in the "Room Manage" page type. Administration pages are located in the "Room Admin" page type. In addition you can of course also change access restrictions for each of the custom databases.
